Detailed Description

template<typename T, std::size_t Rows, std::size_t Columns>
class numpp::matrix::sparse::block< T, Rows, Columns >

Block version of sparse matrix.

Template Parameters
Tarithmetic type contained in matrix class
Rowsnumber of rows in matrix
Columnsnumber of columns in matrix

This implementation is an experiment in cache hit-rate.
 Whole sparse matrix is contained in one array, divided based on blocks of contiguous non-zero numbers.
After each block operation calculating row is performed.

Warning
For parallelization of the sparse matrix dense vector multiplication you need compiler with OpenMP support
Only sparse matrix dense vector multiplication implemented
Each row has to be seperate std::initializer_list

Pros:

  • Obviously lower memory overhead, for 50% sparse matrix with no blocks it contains only 150% storage of dense matrix.
  • Higher cache locality (at least in theory)

Cons:

  • Impossible SIMD optimization
  • Additional operations of read for every block

Example:

numpp::matrix::sparse::nested<double, 3,3> mat{{0, 0, 1},
{0, 2.3, 0},
{0, 0, 13.7}
};
constexpr numpp::matrix::vector<double, 3> vec{1,2,3};
auto result = mat*vec;
